Architecture

Design system + theme layer

The core provides structure — grid, spacing, typography, components. Theme files override the tokens and optionally restyle components. One extra <link> tag is all it takes.

⬡

@layer Architecture

Cascade Layers give each file a defined specificity slot. Themes override components without !important.

◈

oklch Colors

All palette tokens use oklch(). Tints and states via color-mix() — no Sass, no preprocessor.

⟐

CSS Nesting

Native & nesting keeps component styles co-located. Readable and refactorable.

◉

Dark-first Theming

Dark by default. Switch via data-theme="light" or auto via prefers-color-scheme.

▲

Fluid Typography

All font sizes use clamp(). Scales fluidly — no breakpoint overrides needed.

⬡

Splittable Files

Import only what you need, or run node build.js to bundle everything into a single dist file.

Usage

Start in 30 seconds

One design system, one theme. No build step. No dependencies.

1. Load the design system

HTML 
<!-- Design system core -->
<link rel="stylesheet"
      href="arc.css" />

<!-- Pick a theme -->
<link rel="stylesheet"
      href="themes/theme-cyber.css"
      />

2. Compose with classes

HTML 
<div class="container">
  <div class="grid-auto-md">
    <div class="card card-hover">
      <h3 class="card-title">
        Title</h3>
    </div>
  </div>
</div>

3. Override any token

CSS 
/* Custom theme extension */
@layer arc.theme {
  :root {
    --primary: oklch(70% 0.25 30);
    --font-display: "My Font";
    --r-md: 0.75rem;
  }
}

4. Build for production

Shell 
# Bundles all files into dist/
node build.js

Compatibility

Browser support

ARC CSS uses @layer, oklch(), color-mix(), CSS Nesting, clamp(), and @property. CSS Nesting is the limiting factor — ~96% global coverage.
